# Pesquisa - Deploy e Infraestrutura Cloud

## ❓ Pergunta central
Quais são as melhores práticas e ferramentas para deploy e gestão de infraestrutura em Cloud?

## 🎯 Síntese (3-5 linhas)
O deploy e a infraestrutura em cloud permitem maior escalabilidade, flexibilidade e automação em aplicações modernas. Ferramentas como Kubernetes, Terraform e CI/CD pipelines são essenciais. As melhores práticas incluem automação de processos, monitoramento contínuo e uso de padrões como IaC (Infraestrutura como Código). A escolha da solução depende de fatores como custo, complexidade e requisitos de conformidade.

## 🔬 Detalhes
- Infraestrutura como Código (IaC) com ferramentas como Terraform e AWS CloudFormation permite definir e gerenciar recursos em cloud de forma declarativa.
- Kubernetes é amplamente utilizado para orquestração de contêineres, garantindo escalabilidade e alta disponibilidade.
- Pipelines de CI/CD (como GitHub Actions e Jenkins) automatizam o build, teste e deploy, reduzindo o tempo de entrega de novas funcionalidades.
- Monitoramento contínuo (com ferramentas como Prometheus e Datadog) é crucial para identificar falhas e otimizar a performance.
- A escolha de provedores de cloud (AWS, Azure, GCP) deve considerar custo, serviços oferecidos, compliance e suporte.
- Estratégias de segurança, como políticas de IAM (Identity and Access Management) e criptografia de dados, são fundamentais para proteger a infraestrutura.
