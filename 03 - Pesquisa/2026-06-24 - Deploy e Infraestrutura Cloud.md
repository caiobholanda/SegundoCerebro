# Pesquisa - Deploy e Infraestrutura Cloud

## ❓ Pergunta central
Como implementar e gerenciar uma infraestrutura cloud eficiente e realizar deploys de forma escalável e segura?

## 🎯 Síntese (3-5 linhas)
Deploy e infraestrutura cloud são pilares fundamentais para a entrega de software moderno. As melhores práticas envolvem automação, escalabilidade, segurança e monitoramento contínuo. A adoção de ferramentas como CI/CD, containers (Docker) e orquestração (Kubernetes) são estratégias indispensáveis para gerenciar ambientes na nuvem. Além disso, o uso de provedores como AWS, Azure e GCP permite flexibilidade e otimização de custos, mas exige conhecimento técnico para evitar armadilhas.

## 🔬 Detalhes
- **Automação:** Ferramentas de CI/CD (ex.: Jenkins, GitHub Actions) automatizam o processo de build, teste e deploy, reduzindo erros humanos.
- **Containers e Orquestração:** Docker e Kubernetes são amplamente usados para padronizar ambientes e gerenciar aplicações em grande escala.
- **Provedores Cloud:** AWS, Azure e Google Cloud oferecem recursos como compute, storage e networking, mas com diferentes modelos de custo e serviços adicionais.
- **Infraestrutura como Código (IaC):** Ferramentas como Terraform e AWS CloudFormation facilitam a definição e a replicação de infraestruturas.
- **Segurança:** Práticas como controle de acesso baseado em funções (RBAC), criptografia de dados em trânsito e em repouso, além da conformidade com normas, são críticas.
- **Monitoramento e Logging:** Soluções como Prometheus, Grafana e ELK Stack permitem observabilidade e ajudam a identificar problemas antes que impactem os usuários.
