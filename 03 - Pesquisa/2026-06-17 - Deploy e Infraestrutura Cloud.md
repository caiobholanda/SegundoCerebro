# Pesquisa - Deploy e Infraestrutura Cloud

## ❓ Pergunta central
Quais são as melhores práticas e ferramentas para realizar deploys eficientes e gerenciar infraestrutura em nuvem?

## 🎯 Síntese (3-5 linhas)
Deploy e infraestrutura em nuvem são pilares fundamentais para modernizar aplicações e aumentar a escalabilidade. As melhores práticas incluem automação, monitoramento contínuo e uso de ferramentas como Terraform, Kubernetes e Docker. A escolha de provedores (como AWS, Azure ou GCP) deve considerar custo, performance e suporte. A segurança de dados e a gestão eficiente de custos são desafios críticos.

## 🔬 Detalhes
- **Infraestrutura como código (IaC)**: Ferramentas como Terraform e AWS CloudFormation permitem descrever e gerenciar infraestrutura de forma declarativa, promovendo consistência e automação.
- **Contêineres e Orquestração**: Docker facilita a criação de ambientes isolados para aplicações, enquanto Kubernetes gerencia esses contêineres em escala.
- **CICD Pipelines**: Integração e entrega contínuas são essenciais para automação de deploys, com ferramentas como Jenkins, GitHub Actions e GitLab CI.
- **Segurança em nuvem**: Inclui práticas como controle de acesso baseado em funções (IAM), criptografia de dados em repouso e em trânsito, além de monitoramento de vulnerabilidades.
- **Escalabilidade elástica**: Provedores como AWS, Azure e GCP oferecem recursos escaláveis automaticamente, permitindo lidar com picos de demanda sem intervenção manual.
- **Gestão de custos**: Ferramentas como AWS Cost Explorer e GCP Billing Report ajudam a monitorar e otimizar o gasto com recursos.
