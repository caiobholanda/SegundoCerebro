# Pesquisa - Deploy e Infraestrutura Cloud

## ❓ Pergunta central
Como selecionar e configurar uma infraestrutura cloud eficiente para deploys modernos?

## 🎯 Síntese (3-5 linhas)
Deploys e infraestrutura cloud são pilares da entrega contínua de software. Escolher a plataforma certa depende de fatores como custo, escalabilidade, segurança e necessidade de integração com outros serviços. Tecnologias como containers, Kubernetes e práticas de CI/CD são indispensáveis para criar soluções robustas e ágeis, enquanto o uso de provedores como AWS, Azure e GCP oferece flexibilidade e suporte a diferentes demandas de negócio.

## 🔬 Detalhes
- **Provedores populares**: AWS, Azure e Google Cloud Platform (GCP) lideram o mercado, oferecendo serviços amplos e escaláveis.
- **Containers como padrão**: Ferramentas como Docker ajudam a criar ambientes consistentes para aplicações, independentemente da infraestrutura.
- **Orquestração com Kubernetes**: Kubernetes é amplamente adotado para gerenciar containers em escala, automatizando deploys e balanceamento de carga.
- **Práticas de CI/CD**: Integração Contínua e Deploy Contínuo permitem entregas frequentes e com menos erros, utilizando ferramentas como Jenkins, GitLab CI ou GitHub Actions.
- **Infraestrutura como Código (IaC)**: Tecnologias como Terraform e AWS CloudFormation facilitam a configuração e o gerenciamento de infraestrutura.
- **Custo e otimização**: O uso de instâncias sob demanda, serverless e escalabilidade automática (auto-scaling) são estratégias eficazes para otimizar custos em ambientes cloud.
