# Pesquisa - Deploy e Infraestrutura Cloud

## ❓ Pergunta central
Como planejar e executar um deploy eficiente utilizando infraestrutura cloud de forma escalável e segura?

## 🎯 Síntese (3-5 linhas)
Deploy e infraestrutura cloud são pilares para aplicações modernas. Utilizar provedores como AWS, GCP e Azure permite criar ambientes escaláveis, resilientes e flexíveis. A escolha dos serviços adequados, o uso de práticas como IaC (Infrastructure as Code) e a adoção de pipelines CI/CD são fundamentais para automatizar e simplificar processos. No entanto, a segurança e a otimização de custos precisam ser cuidadosamente monitoradas.

## 🔬 Detalhes
- **Provedores de Cloud**: AWS, Google Cloud Platform (GCP) e Microsoft Azure lideram o mercado, oferecendo uma vasta gama de serviços como computação, armazenamento e redes.
- **Infrastructure as Code (IaC)**: Ferramentas como Terraform e AWS CloudFormation permitem gerenciar a infraestrutura como código, garantindo reprodutibilidade e versionamento.
- **Pipelines CI/CD**: Integração contínua e deploy contínuo (CI/CD) são essenciais para automatizar testes e entregas. Ferramentas como Jenkins, GitHub Actions e GitLab CI/CD são amplamente utilizadas.
- **Escalabilidade**: A infraestrutura cloud permite escalabilidade horizontal (adicionar mais máquinas) e vertical (aumentar recursos de uma máquina), ajustando-se às demandas.
- **Segurança**: Implementar boas práticas de segurança como controle de acessos, monitoramento contínuo e uso de firewalls (ex.: AWS WAF) é essencial para proteger os dados e aplicações.
- **Custo**: Gerenciar custos na cloud requer otimização de recursos, uso de instâncias reservadas/spot e monitoramento com ferramentas como AWS Cost Explorer.
