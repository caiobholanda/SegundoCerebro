# Pesquisa - Deploy e Infraestrutura Cloud

## ❓ Pergunta central
Quais são as melhores práticas e ferramentas para realizar deploy e gerenciar infraestrutura em cloud?

## 🎯 Síntese (3-5 linhas)
Deploy e gerenciamento de infraestrutura em cloud envolvem ferramentas e práticas que garantem alta disponibilidade, escalabilidade e segurança. O uso de pipelines CI/CD automatiza o processo de entrega, enquanto plataformas como AWS, Azure e GCP permitem customizar recursos e gerenciar aplicações. A infraestrutura como código (IaC) e o monitoramento contínuo são essenciais para eficiência e prevenção de problemas.

## 🔬 Detalhes
- **Infraestrutura como Código (IaC)**: Ferramentas como Terraform e AWS CloudFormation permitem descrever a infraestrutura em arquivos de configuração, facilitando replicação e versionamento.
- **Plataformas líderes**: Amazon Web Services (AWS), Microsoft Azure e Google Cloud Platform (GCP) oferecem serviços de deploy, armazenamento, escalabilidade e gerenciamento de recursos.
- **CI/CD Pipelines**: Integração e entrega contínua (CI/CD) utilizam ferramentas como GitHub Actions, Jenkins, e GitLab CI para automatizar testes e deploys.
- **Kubernetes e Contêineres**: Kubernetes é amplamente usado para orquestrar contêineres em clusters, garantindo alta disponibilidade e fácil escalabilidade.
- **Monitoramento e Logs**: Ferramentas como Prometheus, Grafana e AWS CloudWatch são cruciais para observar a saúde da aplicação e diagnosticar problemas.
- **Segurança na Cloud**: Práticas como controle de acesso granular, criptografia de dados e uso de redes privadas virtuais (VPNs) são essenciais.
