# Pesquisa - Deploy e Infraestrutura Cloud

## ❓ Pergunta central
Como implementar uma infraestrutura cloud escalável e resiliente para suportar deployments contínuos de aplicações modernas?

## 🎯 Síntese (3-5 linhas)
Deploy e infraestrutura cloud são pilares para a entrega ágil de software. O uso de práticas como CI/CD, containers, orquestração com Kubernetes e ferramentas de IaC (Infraestrutura como Código) tornou possível construir sistemas escaláveis e resilientes. Para maximizar os benefícios, é essencial entender conceitos de automação, segurança e otimização de custos.

## 🔬 Detalhes
- **Infraestrutura como Código (IaC)**: Ferramentas como Terraform e AWS CloudFormation permitem gerenciar infraestrutura de forma declarativa, garantindo consistência e rastreabilidade.
- **Containers e Orquestração**: Docker e Kubernetes possibilitam empacotar e gerenciar aplicações de forma portátil e escalável.
- **CI/CD (Integração Contínua/Entrega Contínua)**: Pipelines automatizados, como os configurados no GitHub Actions ou Jenkins, aceleram o ciclo de desenvolvimento e entrega.
- **Escalabilidade e Resiliência**: Estratégias como auto-scaling, load balancing e failover garantem alta disponibilidade e desempenho em picos de demanda.
- **Segurança na Nuvem**: Políticas de IAM (Identity and Access Management), criptografia de dados e monitoramento contínuo são cruciais para mitigar riscos.
- **Custo-eficiência**: Monitorar e otimizar o uso de recursos cloud, utilizando ferramentas nativas dos provedores (AWS Cost Explorer, Azure Cost Management) ou de terceiros, reduz gastos desnecessários.
