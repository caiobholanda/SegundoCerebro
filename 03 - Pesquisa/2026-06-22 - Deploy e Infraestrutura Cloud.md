# Pesquisa - Deploy e Infraestrutura Cloud

## ❓ Pergunta central
Como configurar, gerenciar e otimizar deploys e infraestrutura em ambientes cloud modernos?

## 🎯 Síntese (3-5 linhas)
O deploy e a infraestrutura cloud são pilares fundamentais para a entrega ágil de software. Com o uso de ferramentas como Kubernetes, Terraform e serviços de nuvem como AWS, Azure e GCP, é possível automatizar processos, escalar aplicações e reduzir custos. A adoção de pipelines CI/CD é essencial para garantir eficiência no ciclo de desenvolvimento. A segurança e o monitoramento contínuo são indispensáveis para a confiabilidade do sistema.

## 🔬 Detalhes
- **Deploy automatizado**: Ferramentas como Jenkins, GitHub Actions e GitLab CI/CD ajudam a criar pipelines para deploy contínuo, reduzindo erros manuais.
- **Infraestrutura como código (IaC)**: Soluções como Terraform e AWS CloudFormation permitem configurar e gerenciar recursos de infraestrutura de forma declarativa.
- **Contêineres e orquestração**: Kubernetes é amplamente utilizado para gerenciar contêineres em escala, garantindo alta disponibilidade e resiliência.
- **Escalabilidade horizontal e vertical**: Serviços como AWS Auto Scaling e Kubernetes Horizontal Pod Autoscaler ajustam dinamicamente a infraestrutura conforme a demanda.
- **Monitoramento e logging**: Ferramentas como Prometheus, Grafana e ELK Stack são essenciais para rastrear métricas e logs em tempo real.
- **Custos e otimização**: A implementação de práticas como Reserved Instances (AWS), otimização de recursos e uso de serviços gerenciados pode reduzir custos operacionais.
