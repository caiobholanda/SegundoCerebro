# Programação e Dev — Novidades de 2026-05-20

## Ranking de Linguagens em 2026

### Top linguagens por adoção/demanda

1. **TypeScript** — #1 no GitHub desde ago/2025 (ultrapassou Python e JavaScript)
   - Usado em: frontend (React, Next.js), backend (Node.js), tooling
   - Razão do domínio: type safety, DX superior, adoção massiva por grandes projetos
   
2. **Python** — dominante em IA/ML
   - Alimenta >80% dos projetos de ML e IA atualmente
   - Core skill para: cientistas de dados, engenheiros de IA, backend
   - Cresce junto com o boom de IA
   
3. **JavaScript** — essencial para web
   - Continua indispensável para frameworks React e Vue.js
   - Runtime cada vez mais: Node.js, Deno, Bun

4. **Rust** — segurança + performance
   - Mais admirada por desenvolvedores (pesquisas Stack Overflow consecutivas)
   - Cresce em: WebAssembly, sistemas embarcados, CLI tools, infraestrutura
   - Adotado por: Linux kernel, Windows, Android, ferramentas de build (Rolldown, Oxc)

5. **Go** — microserviços
   - Simplicidade e eficiência em arquiteturas distribuídas
   - Forte em: backends de alta carga, CLIs, ferramentas DevOps
   - Kubernetes e grande parte da infra cloud escrita em Go

### Linguagens com momentum específico
- **WASM (WebAssembly)**: crescendo como target de compilação para Rust/C++ no browser
- **Kotlin**: domina Android, avança em backend (Kotlin Multiplatform)
- **Swift**: iOS/macOS, avança em server-side

## Grandes Tendências 2026

### 1. IA como Infraestrutura de Dev
A maior mudança: toda pessoa desenvolvedora usa IA como ferramenta padrão.
- **GitHub Copilot**, **CodeWhisperer**, **Claude Code** — onipresentes
- Papel do dev mudando de "escritor de código" para **revisor e arquiteto**
- LLMs como layer de abstração para geração, refatoração e debugging

### 2. DevSecOps como Padrão
- Segurança integrada no pipeline desde o primeiro commit
- SAST/DAST automático em CI/CD
- Shift-left: vulnerabilidades detectadas em desenvolvimento, não em produção
- Compliance como código (OPA, Conftest)

### 3. Green Software / Programação Sustentável
- Tendência emergente: código eficiente para reduzir consumo de energia
- Métricas de carbon footprint no CI/CD
- Preferência por algoritmos O(n log n) sobre O(n²) como critério de code review
- Cloud regions com energia renovável como critério de arquitetura

### 4. Low-Code Complementar
- Low-code não substitui devs — complementa para automações e protótipos
- Devs como "orquestradores" de ferramentas low-code + código customizado

## Papel da IA no Dev em 2026

```
Antes: Dev → escreve código → testa → revisa
Agora: Dev → especifica intenção → AI escreve → Dev revisa + arquiteta → testa
```

Habilidades valorizadas que IA não substitui:
- Definição de arquitetura e trade-offs
- Revisão crítica de código gerado
- Debugging de problemas complexos de runtime
- Comunicação com stakeholders
- Entendimento de domínio de negócio
