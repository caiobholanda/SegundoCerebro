# Pesquisa - Deploy e Infraestrutura Cloud

## ❓ Pergunta central
Como criar e gerenciar uma infraestrutura cloud eficiente e otimizar processos de deploy?

## 🎯 Síntese (3-5 linhas)
Deploy e infraestrutura cloud são pilares fundamentais para a entrega ágil de software. A escolha da arquitetura correta, aliada ao uso de ferramentas de automação e práticas de DevOps, garante escalabilidade, alta disponibilidade e segurança. Compreender os serviços oferecidos por provedores como AWS, Azure e Google Cloud é essencial para construir ambientes robustos e econômicos.

## 🔬 Detalhes
- A infraestrutura cloud permite escalabilidade elástica, atendendo picos de demanda sem necessidade de hardware adicional.
- Deploy contínuo (CD) automatiza o processo de entrega de código, minimizando erros humanos e acelerando lançamentos.
- Provedores populares como AWS, Azure e Google Cloud oferecem serviços gerenciados como Kubernetes (EKS, AKS, GKE) para orquestração de contêineres.
- Infraestrutura como Código (IaC) usando ferramentas como Terraform ou AWS CloudFormation facilita o gerenciamento e versionamento de recursos cloud.
- Monitoramento e observabilidade são fundamentais para identificar problemas e otimizar a performance da infraestrutura.
- Design de arquiteturas resilientes, como autoscaling, load balancing e redundância geográfica, garante alta disponibilidade.
