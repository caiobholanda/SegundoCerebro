# Pesquisa - Deploy e Infraestrutura Cloud

## ❓ Pergunta central
Como estruturar um processo eficiente de deploy utilizando infraestrutura cloud moderna?

## 🎯 Síntese (3-5 linhas)
Deploy e infraestrutura cloud representam pilares fundamentais para entrega ágil e escalável de software. O uso de práticas como CI/CD, orquestração de containers e computação serverless são essenciais para maximizar eficiência e minimizar erros. Além disso, a escolha entre provedores como AWS, Azure e Google Cloud deve considerar custo, escalabilidade e necessidades específicas do projeto.

## 🔬 Detalhes
- **Infraestrutura como Código (IaC)**: Ferramentas como Terraform e AWS CloudFormation permitem definir e gerenciar infraestrutura de forma programática, garantindo reprodutibilidade e controle de versões.
- **CI/CD (Integração e Deploy Contínuos)**: Automação do ciclo de desenvolvimento, possibilitando testes e deploy frequentes com ferramentas como Jenkins, GitHub Actions e GitLab CI/CD.
- **Containerização**: Docker e Kubernetes são padrões para empacotar e orquestrar aplicações, garantindo portabilidade e escalabilidade.
- **Serverless Computing**: Serviços como AWS Lambda e Google Cloud Functions permitem executar código sem gerenciar servidores, ideal para cargas dinâmicas e eventos pontuais.
- **Monitoramento e Observabilidade**: Ferramentas como Prometheus, Grafana e Datadog são cruciais para identificar gargalos e assegurar que a infraestrutura opere conforme esperado.
- **Provedores Cloud**: AWS, Azure e Google Cloud dominam o mercado, mas provedores menores como DigitalOcean podem ser vantajosos para projetos específicos devido a simplicidade e custo.
