# Pesquisa - Deploy e Infraestrutura Cloud

## ❓ Pergunta central
Como realizar um deploy eficiente e seguro em uma infraestrutura cloud moderna?

## 🎯 Síntese (3-5 linhas)
A infraestrutura cloud transformou o processo de deploy, trazendo escalabilidade, flexibilidade e automação. No entanto, práticas como CI/CD, orquestração de contêineres e otimização de custos são essenciais para maximizar os benefícios. Além disso, a segurança deve ser uma prioridade desde o design da aplicação até o monitoramento contínuo.

## 🔬 Detalhes
- O deploy em ambientes cloud é altamente dependente de ferramentas de automação como Jenkins, GitHub Actions e GitLab CI/CD.
- Infraestruturas como código (IaC), usando ferramentas como Terraform e AWS CloudFormation, permitem gerenciar recursos de forma declarativa e escalável.
- Contêineres (Docker) e orquestradores (Kubernetes) são amplamente utilizados para simplificar o gerenciamento de aplicativos em ambientes distribuídos.
- O uso de microsserviços é uma prática comum, permitindo que aplicações sejam divididas em componentes menores, independentes e escaláveis.
- A segurança é crítica, incluindo práticas como gerenciamento de segredos, controle de acesso baseado em funções (RBAC) e auditorias regulares de infraestrutura.
- Custos podem ser gerenciados usando políticas de escalonamento automático, instâncias spot e monitoramento contínuo de recursos para evitar desperdícios.
