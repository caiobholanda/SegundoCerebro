# Pesquisa - Claude Code — CLI e agentes

## ❓ Pergunta central
Como o Claude Code utiliza CLI e agentes para orquestrar tarefas complexas no Second Brain?

## 🎯 Síntese (3-5 linhas)
O Claude Code aproveita a interface de linha de comando (CLI) para executar tarefas automatizadas e interagir com o Second Brain de forma programática. Agentes personalizados são criados para resolver problemas específicos, atuando como extensões que compreendem e manipulam dados estruturados no Obsidian. Essa sinergia permite maior produtividade e flexibilidade na gestão do conhecimento.

## 🔬 Detalhes
- O Claude Code utiliza CLI para integrar o Obsidian com ferramentas externas, como geradores de texto e scripts de automação, aumentando a eficiência do sistema.
- Agentes são implementados para realizar tarefas específicas, como criação de notas, organização de pastas e análise de dados, com base em prompts personalizados.
- A integração entre CLI e agentes permite o uso de comandos simples para acionar fluxos de trabalho complexos no Second Brain.
- O sistema é projetado para ser modular, permitindo que novos agentes sejam facilmente adicionados ou removidos conforme necessário.
- A estrutura YAML no topo das notas é essencial para que os agentes entendam o contexto e manipulem as informações adequadamente.
- A segurança é um ponto crítico: o Claude Code opera dentro de um sandbox no Obsidian para evitar interações não autorizadas com o sistema ou dados sensíveis.
