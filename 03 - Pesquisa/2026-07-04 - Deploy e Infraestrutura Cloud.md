# Pesquisa - Deploy e Infraestrutura Cloud

## ❓ Pergunta central
Quais são as melhores práticas e ferramentas atuais para realizar deploys eficientes e gerenciar infraestrutura em ambientes de cloud computing?

## 🎯 Síntese (3-5 linhas)
A infraestrutura cloud é vital para a escalabilidade e resiliência de aplicações modernas. Boas práticas incluem o uso de automação com ferramentas como Terraform e Ansible, adoção de estratégias como infraestrutura imutável e pipelines CI/CD, além de monitoramento contínuo. A escolha da cloud provider e o uso de containers e orquestradores como Kubernetes são decisivos para o sucesso de projetos na nuvem.

## 🔬 Detalhes
- A infraestrutura como código (IaC) é essencial para gerenciar e versionar recursos, com ferramentas como [[Terraform]] e [[AWS CloudFormation]] liderando o mercado.
- O uso de containers, especialmente com [[Docker]], e sua orquestração com [[Kubernetes]] são padrões de mercado para garantir portabilidade, escalabilidade e isolamento.
- Estratégias como **infraestrutura imutável** (onde os servidores não são alterados após o deploy) evitam problemas de configuração e facilitam rollback.
- Pipelines de CI/CD, usando ferramentas como [[Jenkins]], [[GitHub Actions]] e [[GitLab CI]], automatizam testes, builds e deploys, reduzindo erros manuais.
- O monitoramento contínuo é crítico, com ferramentas como [[Prometheus]] e [[Datadog]] sendo amplamente utilizadas para garantir a disponibilidade e desempenho.
- A escolha entre diferentes provedores de cloud (como [[AWS]], [[Google Cloud]], [[Microsoft Azure]]) depende de fatores como custo, compatibilidade tecnológica e suporte a serviços gerenciados específicos.
