# Pesquisa - Claude Code — CLI e agentes

## ❓ Pergunta central
Como o Claude Code utiliza CLI e agentes para expandir suas capacidades e integrá-las com fluxos de trabalho?

## 🎯 Síntese (3-5 linhas)
O Claude Code combina uma interface de linha de comando (CLI) robusta com agentes que permitem automação e integração entre sistemas diversos. Por meio do CLI, é possível interagir diretamente com o modelo para realizar tarefas customizadas. Os agentes, por sua vez, são scripts programáveis que atuam como intermediários em fluxos automatizados, otimizando produtividade em aplicações como gestão de dados, geração de conteúdo e suporte técnico.

## 🔬 Detalhes
- O **CLI do Claude Code** serve como uma interface para executar comandos diretamente no modelo, permitindo integração com pipelines e sistemas de automação.
- **Agentes no Claude Code** são instâncias que conseguem executar tarefas específicas, como extração e manipulação de dados ou interações com APIs externas.
- A configuração de agentes é feita via arquivos JSON ou YAML, que definem as funções, permissões e contextos nos quais o agente deve operar.
- A segurança é central: os agentes do Claude Code operam sob um modelo de permissões restritas, exigindo autenticação para executar operações sensíveis.
- Existe integração nativa do CLI com ferramentas populares como Git, Docker e AWS, permitindo fluxos de trabalho contínuos para desenvolvedores.
- Um dos casos de uso mais populares é a criação de pipelines de CI/CD, onde agentes automatizam tarefas como revisão de código, testes e deploys.
