# Referência — GreenOps: Por Que é o Novo Padrão em 2026

## Ferramentas de Medição de Consumo de Energia

| Ferramenta | Uso principal | Integração |
|------------|--------------|-----------|
| **Scaphandre** | Consumo energético em containers | Docker/Kubernetes |
| **Greenspector** | Monitoramento contínuo em CI/CD | Pipelines CI/CD |
| **CodeCarbon / CarbonTracker** | Pegada de carbono em ML | Python, TensorFlow, PyTorch |
| **Carbon Aware SDK (GSF)** | Agendamento consciente de carbono | Cloud workloads |

## Framework SOFT

Ratificado pela Green Software Foundation em novembro de 2025:
- Do nível executivo até práticas diárias de dev
- Checklists de revisão de código com dimensão carbônica
- Estratégias de deployment em períodos de maior energia limpa
- Procurement orientado por carbono

## Métrica SCI (Software Carbon Intensity)

Padrão ISO-certificado da GSF:
- Mede emissões por unidade de trabalho (por API call, transação ou sessão de usuário)
- Torna carbono mensurável e otimizável no ciclo de dev

## Dados de impacto

- Algoritmos mais eficientes reduzem consumo de energia em 30-95%
- Caching efetivo reduz tempo de computação em 50-80%
- Lazy loading diminui uso de memória em 20-40%
- C é 75x mais eficiente energeticamente que Python

## MCP Server relacionado

"Software Energy Efficiency MCP Server" — ajuda devs a escrever código energeticamente eficiente durante o desenvolvimento.
